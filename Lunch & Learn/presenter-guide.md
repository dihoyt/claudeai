### Slide 5-6: What Made This Effective (10 minutes)
**Key Message:** "Context + Constraints + Trust = Results"

**Delivery Tips:**
- Use the split: what user provided vs what AI contributed
- Emphasize collaboration, not dictation
- This is the core teaching moment

**Interactive Element:**
Ask participants:
- "What would YOU have added to these prompts?"
- Most will say things that weren't needed
- Discussion point: less is more with context

**Watch For:**
- Participants who want to over-specify
- Address: "Trust the AI to apply expertise"

### Slide 7-11: Core Principles (10 minutes)
**Key Message:** "Change how you think about working with AI"

**Delivery Tips:**
- Give examples for each principle
- Use the ‚ùå/‚úÖ format - clear contrast
- Encourage note-taking here

**Principle 1: Context over Commands**
- Bad: "Create a prompt"
- Good: "Create a prompt for project managers to scope new IT initiatives for senior stakeholders"

**Principle 2: Constraints over Details**
- Constraints guide, details restrict
- "Assume MS 365 access" vs "Query this specific API endpoint..."

**Principle 3: Trust**
- Hardest for many people
- Address: "You can always refine"

**Principle 4: Iterate**
- Show conversation flow from repo
- Natural progression

### Slide 22-23: **HANDS-ON EXERCISE** (15 minutes)
**Key Message:** "Now it's your turn"

**Exercise Structure:**

**1. Setup (2 minutes)**
- Show the template
- Explain the task
- Set 10-minute timer

**2. Individual Work (10 minutes)**
- Participants draft a prompt
- Use template provided
- You circulate and help

**3. Share & Discuss (3 minutes)**
- 2-3 volunteers share
- Quick feedback
- Identify what worked

**Template to Share:**
```markdown
# [Agent Name]
## Purpose
[What problem does this solve?]
## Context
- Data source/tool 1
- Data source/tool 2
- Key constraints
## Instructions
1. Step one
2. Step two
3. Final output
## Output Format
[Describe what you want to receive]
```

**Facilitation Tips:**
- Have a few example scenarios ready if people get stuck
- Encourage thinking about repetitive tasks
- It's okay if they don't finish - the process matters

**Example Scenarios to Suggest:**
- Meeting notes summarizer
- Email triage assistant
- Code documentation generator
- Weekly status report creator

### Slide 45-47: Q&A (5-10 minutes)
**Key Message:** "Let's discuss"

**Common Questions & Answers:**

**Q: "Will this replace my job?"**
A: No, it augments your work. You still provide judgment, strategy, creativity. AI handles repetitive structure.

**Q: "What if it gives wrong information?"**
A: Always review outputs. AI is a draft generator, you're the editor.

**Q: "How do I know if my prompt is good?"**
A: Test it! If 80% of outputs are useful, it's good.

**Q: "Can I use this for [specific task]?"**
A: If it's structured and repeatable, probably yes. Try it!

**Q: "What about data privacy?"**
A: Use your organization's approved AI tools. Don't put sensitive data in prompts.

**Q: "How much does this cost?"**
A: Depends on tool. Claude.ai has free tier. API has usage costs. Usually worth the time savings.

## üé≠ Presentation Tips

### Energy & Pacing:
- Start strong with the big numbers
- Build momentum through principles
- Peak energy during hands-on exercise
- Close inspirationally

### Engagement Techniques:
- Ask questions throughout
- Call on volunteers (gently)
- Share personal examples
- Acknowledge skepticism when you see it

### Handling Different Audiences:

**Technical Audience:**
- Spend more time on integration
- Show API examples
- Discuss architecture

**Business Audience:**
- Focus on ROI and time savings
- More real-world examples
- Less technical detail

**Mixed Audience:**
- Balance both
- Offer "for more technical details, see me after"

### If Running Long:
**Can Skip:**
- Slides 15-17 (Productivity analysis - numbers speak for themselves)
- Slides 28-30 (Common pitfalls - share as a reference)
- Slide 21 (Meta moment - fun but not essential)

**Can't Skip:**
- The conversation (Slides 3-4)
- Core principles (Slides 7-11)
- Hands-on exercise (Slide 22-23)
- Q&A (Slide 45-47)

### If Running Short:
**Can Expand:**
- Live demo with audience suggestions
- More time for hands-on exercise
- Extended Q&A
- Share additional examples

## üí° Pro Tips

### Before You Start:
1. **Test your demo** - Murphy's law applies
2. **Have backup slides** - PDF in case markdown viewer fails
3. **Know your audience** - adjust technical depth
4. **Prepare 3 good examples** - relevant to their work

### During Presentation:
1. **Read the room** - adjust pace based on engagement
2. **Embrace questions** - they mean people care
3. **Share failures** - what didn't work for you
4. **Be authentic** - you're learning too

### Common Challenges:

**"This seems too good to be true"**
- Show howitsmade.md
- Acknowledge it's shocking
- Emphasize the method, not magic

**"I tried AI and it didn't work"**
- Ask about their approach
- Compare to principles taught
- Offer to help debug

**"We can't use AI tools at work"**
- Respect constraints
- Share principles still apply
- Suggest approved alternatives

**Technical issues during demo**
- Have screenshots ready
- Fall back to showing files
- Make a joke, move on
